# Captain 0.1.0-alpha.7

Captain's seventh public early-access release strengthens the local runtime
boundary: committed state survives abrupt process loss, macOS services restart
after unexpected exits, model context follows the active provider catalog, and
kernel-backed tools remain available when the TUI or CLI runs Captain in
process. Telegram Rich Messages and the operational contracts from alpha.6
remain unchanged.

## Install

macOS, Linux, and VPS:

```bash
curl -fsSL https://github.com/Vivien83/captain/releases/download/v0.1.0-alpha.7/install.sh \
  | CAPTAIN_VERSION=v0.1.0-alpha.7 bash
```

Docker (`linux/amd64` and `linux/arm64`):

```bash
docker pull ghcr.io/vivien83/captain-agent-os:v0.1.0-alpha.7
```

The moving prerelease channel is
`ghcr.io/vivien83/captain-agent-os:alpha`. Pin the immutable version tag for a
reproducible install.

The release also contains an `x86_64-pc-windows-msvc` CLI zip. Every host
archive has a SHA-256 sidecar and platform manifest; `manifest.json` covers all
five targets.

## Highlights

- Direct TUI and CLI streaming turns receive the live in-process kernel handle
  when no daemon is reachable. `memory_save` and other kernel-backed tools keep
  the same runtime contract as Web/API turns.
- Captain confirms a durable memory only after `memory_save` succeeds. Failed
  writes are reported as not stored instead of being acknowledged early.
- An installed macOS LaunchAgent uses `RunAtLoad`, `KeepAlive`, and bounded
  restart throttling. Unexpected exits are supervised; an intentional
  `captain service stop` unloads the job and remains stopped.
- Context capacity is resolved from the configured provider and model on every
  turn, switch, API detail, and restored session. Codex uses its active
  `context_window`; `max_context_window` remains only an override ceiling.
- The TUI footer reports active prompt occupancy against that effective window
  instead of presenting cumulative lifetime tokens as current context.
- SQLite-backed state uses WAL and `synchronous=FULL`. Captain-managed files
  are committed through synchronized sibling replacement plus parent-directory
  synchronization, including `F_FULLFSYNC` on macOS.
- A reproducible full-daemon recovery smoke commits memory, project, session,
  event, and configuration state, sends `SIGKILL`, restarts the same home, and
  verifies every value plus SQLite integrity.
- Fresh-home MemPalace repair completes before the asynchronous daemon runtime
  starts, avoiding the blocking-client runtime-destruction panic seen during
  native bootstrap.

## Recovery boundary

The durability guarantee begins after the relevant tool or API reports success.
An external side effect still in flight at abrupt termination may require
inspection or idempotent retry. Backups remain necessary; no local durability
mechanism makes the host disk itself redundant.

## Known limitation

The explicit per-turn memory write opt-out remains incomplete in `alpha.7`.
Post-turn semantic graph, MemPalace, reflection, conversation-learning, and
workflow-learning writes are suppressed, but the core agent-loop finalizer
still stores its local episodic interaction fragment. This is separate from the
normal session transcript and mandatory audit that are intentionally retained.
Do not rely on the opt-out to prevent that local semantic fragment until a
later immutable release closes the core finalizer path.

## Alpha notice

This is a prerelease. Keep backups, review granted capabilities, and do not use
it as the sole copy of critical data. macOS binaries are ad-hoc signed but not
Apple-notarized; the Windows CLI is not Authenticode-signed. Verify the
published SHA-256 sidecar and expect an explicit operating-system approval on
first launch.

Captain binds to loopback by default. For remote access, configure Captain
authentication and place it behind HTTPS; never expose an unauthenticated
daemon directly to the Internet.

The presentation site is deployed separately at `https://captainagent.fr/`.
Its source is intentionally not part of the public Git repository or release.
