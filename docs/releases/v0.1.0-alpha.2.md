# Captain 0.1.0-alpha.2

Captain's second public early-access release adds native visual inspection to
the active conversation model and aligns every embedded browser surface with
the Captain identity. It retains the durable sessions, detached tools, bounded
memory, guarded autonomy, agent APIs, and authenticated surfaces introduced in
`0.1.0-alpha.1`.

## Install

macOS, Linux, and VPS:

```bash
curl -fsSL https://github.com/Vivien83/captain/releases/download/v0.1.0-alpha.2/install.sh \
  | CAPTAIN_VERSION=v0.1.0-alpha.2 bash
```

Docker (`linux/amd64` and `linux/arm64`):

```bash
docker pull ghcr.io/vivien83/captain-agent-os:v0.1.0-alpha.2
```

The moving prerelease channel is
`ghcr.io/vivien83/captain-agent-os:alpha`. The immutable version tag is
recommended for reproducible installs.

The release also contains an `x86_64-pc-windows-msvc` CLI zip. Every host
archive has a SHA-256 sidecar and platform manifest; `manifest.json` covers all
five targets.

## Highlights

- A prompted `browser_screenshot` is delivered directly to the same active
  multimodal conversation model. Captain does not require a Mistral key, spawn
  a Vision agent, or change provider behind the operator's back.
- Text-only active models refuse image inspection with an actionable model
  switch request instead of making an unsupported visual claim.
- Screenshot pixels are available for the current turn but omitted from
  durable session serialization, preventing restored transcripts from growing
  with base64 payloads.
- Codex and OpenAI-compatible streaming preserve user text, tool results, and
  images in their original order.
- Control, Terminal, Config, Apple touch metadata, and `/favicon.ico` now use
  the same embedded Captain logo.
- Local publication gates can reuse release-profile Rust artifacts, avoiding a
  duplicate debug build tree on constrained maintainer machines.

## Alpha notice

This is a prerelease. Keep backups, review granted capabilities, and do not use
it for critical workloads. macOS binaries are ad-hoc signed but not
Apple-notarized; the Windows CLI is not Authenticode-signed. Verify the
published SHA-256 sidecar and expect an explicit operating-system approval on
first launch.

Captain binds to loopback by default. For remote access, configure Captain
authentication and place it behind HTTPS; never expose an unauthenticated
daemon directly to the Internet.

The presentation site is deployed separately at `https://captainagent.fr/`.
Its source is intentionally not part of the public Git repository or release.
