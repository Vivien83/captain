# Captain 0.1.0-alpha.6

Captain's sixth public early-access release makes Telegram a first-class Rich
Messages surface. Final answers retain structured Markdown, independent tool
calls share a live activity board, long turns use ephemeral drafts instead of
persistent heartbeat noise, and interactive questions remain correct from
prompt through answer or expiry. The production memory, session, model,
agent-API, security, and single-agent first-boot contracts from previous alphas
remain unchanged.

## Install

macOS, Linux, and VPS:

```bash
curl -fsSL https://github.com/Vivien83/captain/releases/download/v0.1.0-alpha.6/install.sh \
  | CAPTAIN_VERSION=v0.1.0-alpha.6 bash
```

Docker (`linux/amd64` and `linux/arm64`):

```bash
docker pull ghcr.io/vivien83/captain-agent-os:v0.1.0-alpha.6
```

The moving prerelease channel is
`ghcr.io/vivien83/captain-agent-os:alpha`. Pin the immutable version tag for a
reproducible install.

The release also contains an `x86_64-pc-windows-msvc` CLI zip. Every host
archive has a SHA-256 sidecar and platform manifest; `manifest.json` covers all
five targets.

## Highlights

- Telegram final answers use Bot API 10.2 Rich Messages and preserve GFM
  tables, lists, code, and collapsible details up to the native Rich limit.
- Private chats receive an ephemeral Rich draft while a response is forming,
  followed by one persistent final message. Groups use the persistent path
  directly.
- Consecutive independent tools share one live activity board. Results remain
  correlated by tool-call id even when they finish out of order; a dependent
  call opens the next board instead of being presented as parallel work.
- Successful tool rows collapse into inspectable details, failures remain open,
  and external tool content is escaped before entering Rich markup.
- After real inactivity, long private turns refresh one ephemeral operational
  draft before Telegram's TTL. Text, tool activity, and visible edits reset the
  timer; no durable "still working" messages are added to the transcript.
- `ask_user` questions are Rich control cards. Button and freeform answers
  reach the active agent loop before the card is confirmed, invalid button
  indices do not consume a question, and every answered or expired card
  explicitly removes its stale keyboard.
- User-visible turn and stale-callback errors are sanitized and rendered as
  Rich cards without provider payloads or secrets.
- Older Bot API servers fall back to the existing Telegram HTML/plain path
  only after an explicit unsupported-endpoint response. Ambiguous network or
  server failures never trigger a second send that could duplicate a message.

## Known limitation

The explicit per-turn memory write opt-out is incomplete in `alpha.6`.
Post-turn semantic graph, MemPalace, reflection, conversation-learning, and
workflow-learning writes are suppressed, but the core agent-loop finalizer
still stores its local episodic interaction fragment. This is separate from the
normal session transcript and mandatory audit that are intentionally retained.
Do not rely on the opt-out to prevent that local semantic fragment until a
later immutable release closes the core finalizer path.

## Alpha notice

This is a prerelease. Keep backups, review granted capabilities, and do not use
it as the sole copy of critical data. macOS binaries are ad-hoc signed but not
Apple-notarized; the Windows CLI is not Authenticode-signed. Verify the
published SHA-256 sidecar and expect an explicit operating-system approval on
first launch.

Captain binds to loopback by default. For remote access, configure Captain
authentication and place it behind HTTPS; never expose an unauthenticated
daemon directly to the Internet.

The presentation site is deployed separately at `https://captainagent.fr/`.
Its source is intentionally not part of the public Git repository or release.
