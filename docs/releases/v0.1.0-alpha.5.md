# Captain 0.1.0-alpha.5

Captain's fifth public early-access release closes seven production reliability
gaps found during live use after `0.1.0-alpha.4`: persistent Web terminal PTYs
now belong to the daemon shutdown lifecycle; an explicit user instruction not
to retain the current message blocks semantic memory writes for that turn; all
prompt profiles receive the configured provider and model; automatic complexity
routing and inferred fallback models are removed; a fresh first boot creates
only the principal `captain` agent; and interrupted release smokes clean their
complete isolated process tree. The managed MemPalace runtime,
authoritative correction flow, native visual inspection, detached tools,
durable sessions, bounded autonomy, agent APIs, and authenticated surfaces
from previous alphas remain available.

## Install

macOS, Linux, and VPS:

```bash
curl -fsSL https://github.com/Vivien83/captain/releases/download/v0.1.0-alpha.5/install.sh \
  | CAPTAIN_VERSION=v0.1.0-alpha.5 bash
```

Docker (`linux/amd64` and `linux/arm64`):

```bash
docker pull ghcr.io/vivien83/captain-agent-os:v0.1.0-alpha.5
```

The moving prerelease channel is
`ghcr.io/vivien83/captain-agent-os:alpha`. The immutable version tag is
recommended for reproducible installs.

The release also contains an `x86_64-pc-windows-msvc` CLI zip. Every host
archive has a SHA-256 sidecar and platform manifest; `manifest.json` covers all
five targets.

## Highlights

- Graceful daemon shutdown drains persistent Web terminal PTYs and terminates
  their owned `captain chat` children before the server exits. Stop/restart no
  longer leaves the prior listener or terminal process alive.
- Explicit French or English instructions not to remember, save, learn, retain,
  or add memory from the current message override remember-like wording in the
  same turn.
- Opted-out turns remain present in the normal conversation transcript and
  mandatory audit trail, while semantic graph facts, MemPalace mirroring,
  reflection, conversation/workflow learning, and other long-term memory writes
  from that turn are suppressed.
- The opt-out is scoped to the current turn and does not silently delete older
  knowledge. Existing facts still use the audited `memory_forget` correction
  path.
- Every prompt profile, including the minimal direct-response path used by the
  TUI, receives the authoritative configured provider and model for the turn.
  Captain no longer infers its own model from old messages or a
  specialist-agent row.
- Captain no longer classifies turns by complexity to replace the configured
  model. Streaming, API/CLI one-shot, Web, Desktop, and messaging-channel turns
  all keep the agent's selected provider/model. A different role or model is an
  explicit sub-agent or model-switch decision.
- New agents have no inferred backup chain. `fallback_models` remain an
  explicit, failure-only operator choice; unrelated API keys on the host do not
  activate another model.
- Fresh setup, init, and factory-reset paths create no specialist agents and do
  not copy the embedded template catalog into runtime state. First boot starts
  with exactly one agent, `captain`; specialists are created explicitly.
- Release-readiness cleanup records native MemPalace bootstrap descendants and
  terminates them before the candidate daemon. A timeout or interrupted smoke
  cannot leave isolated uv/Python workers running after the temporary home is
  removed.

## Alpha notice

This is a prerelease. Keep backups, review granted capabilities, and do not use
it as the sole copy of critical data. macOS binaries are ad-hoc signed but not
Apple-notarized; the Windows CLI is not Authenticode-signed. Verify the
published SHA-256 sidecar and expect an explicit operating-system approval on
first launch.

Captain binds to loopback by default. For remote access, configure Captain
authentication and place it behind HTTPS; never expose an unauthenticated
daemon directly to the Internet.

The presentation site is deployed separately at `https://captainagent.fr/`.
Its source is intentionally not part of the public Git repository or release.
