# Captain 0.1.0-alpha.4

Captain's fourth public early-access release makes explicit user corrections
authoritative throughout memory recall and keeps old active facts discoverable
beyond the recent-write window. It also closes the documented CLI rail for
continuing a persisted agent session by name or UUID. The managed MemPalace
runtime, native visual inspection, detached tools, durable sessions, bounded
autonomy, agent APIs, and authenticated surfaces from previous alphas remain
available.

## Install

macOS, Linux, and VPS:

```bash
curl -fsSL https://github.com/Vivien83/captain/releases/download/v0.1.0-alpha.4/install.sh \
  | CAPTAIN_VERSION=v0.1.0-alpha.4 bash
```

Docker (`linux/amd64` and `linux/arm64`):

```bash
docker pull ghcr.io/vivien83/captain-agent-os:v0.1.0-alpha.4
```

The moving prerelease channel is
`ghcr.io/vivien83/captain-agent-os:alpha`. The immutable version tag is
recommended for reproducible installs.

The release also contains an `x86_64-pc-windows-msvc` CLI zip. Every host
archive has a SHA-256 sidecar and platform manifest; `manifest.json` covers all
five targets.

## Highlights

- The latest user message is authoritative during a correction. Recalled
  memory remains background context and cannot replace exact old or new values
  supplied in the current turn.
- Durable recall searches the complete active local continuity journal rather
  than only the newest writes. Exact active triples are returned before fuzzy
  semantic archives, so a related retraction guard cannot hide a replacement.
- Precise alphanumeric markers outrank generic older corrections, while pure
  dates and numbers are not promoted into misleading identity anchors.
- `memory_save` receipts repeat the bounded object actually stored, making the
  effective subject, predicate, value, and synchronization state auditable
  before Captain confirms success.
- Automatic fact mirroring now applies the same sensitive-field policy as an
  explicit memory write. Verification codes, passwords, tokens, API keys,
  private keys, OTPs, and similarly named secrets are rejected.
- `captain message` accepts a unique agent name or UUID as documented, marks
  one-shot turns as CLI-originated, and preserves the shared persisted session
  catalog used by Web/API and the other Captain surfaces.

## Alpha notice

This is a prerelease. Keep backups, review granted capabilities, and do not use
it as the sole copy of critical data. macOS binaries are ad-hoc signed but not
Apple-notarized; the Windows CLI is not Authenticode-signed. Verify the
published SHA-256 sidecar and expect an explicit operating-system approval on
first launch.

Captain binds to loopback by default. For remote access, configure Captain
authentication and place it behind HTTPS; never expose an unauthenticated
daemon directly to the Internet.

The presentation site is deployed separately at `https://captainagent.fr/`.
Its source is intentionally not part of the public Git repository or release.
