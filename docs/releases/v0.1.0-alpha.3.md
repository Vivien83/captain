# Captain 0.1.0-alpha.3

Captain's third public early-access release makes semantic memory a
self-contained Captain dependency and keeps accepted facts available through
MemPalace outages, process crashes, and restarts. It retains the native visual
inspection, durable sessions, detached tools, bounded autonomy, agent APIs,
and authenticated surfaces from the previous alphas.

## Install

macOS, Linux, and VPS:

```bash
curl -fsSL https://github.com/Vivien83/captain/releases/download/v0.1.0-alpha.3/install.sh \
  | CAPTAIN_VERSION=v0.1.0-alpha.3 bash
```

Docker (`linux/amd64` and `linux/arm64`):

```bash
docker pull ghcr.io/vivien83/captain-agent-os:v0.1.0-alpha.3
```

The moving prerelease channel is
`ghcr.io/vivien83/captain-agent-os:alpha`. The immutable version tag is
recommended for reproducible installs.

The release also contains an `x86_64-pc-windows-msvc` CLI zip. Every host
archive has a SHA-256 sidecar and platform manifest; `manifest.json` covers all
five targets.

## Highlights

- Official host installers and containers provision a Captain-managed
  MemPalace 3.5.0 runtime with pinned uv 0.11.28, isolated CPython 3.13.14,
  and a frozen checksum-bound dependency lock. No system Python, manual
  `pip install`, secondary model provider, or extra API key is required.
- Every active local kernel entrypoint uses the same fail-closed readiness
  preflight. It validates palace access and a real semantic search, repairs
  missing or insecure generations transactionally, and preserves the active
  runtime and user data when repair cannot complete.
- Accepted additions and invalidations are written to a durable local SQLite
  journal before MemPalace synchronization. Local recall therefore continues
  while the semantic index is unavailable.
- Pending and degraded operations recover automatically with persistent,
  bounded exponential backoff. Error rows are never silently age-deleted, and
  one backend outage cannot consume the whole retry queue.
- `memory_forget` retains the original audit row and queues an idempotent remote
  invalidation. Corrections follow a strict retract-old, then save-new order;
  restart reconstruction prevents an old fact from reappearing after a crash.
- `captain doctor` and learning metrics expose backlog state, oldest pending
  operation, next retry, attempt count, and the latest bounded sync error.
  Tool receipts distinguish local durability from confirmed remote indexing.

## Alpha notice

This is a prerelease. Keep backups, review granted capabilities, and do not use
it as the sole copy of critical data. macOS binaries are ad-hoc signed but not
Apple-notarized; the Windows CLI is not Authenticode-signed. Verify the
published SHA-256 sidecar and expect an explicit operating-system approval on
first launch.

Captain binds to loopback by default. For remote access, configure Captain
authentication and place it behind HTTPS; never expose an unauthenticated
daemon directly to the Internet.

The presentation site is deployed separately at `https://captainagent.fr/`.
Its source is intentionally not part of the public Git repository or release.
