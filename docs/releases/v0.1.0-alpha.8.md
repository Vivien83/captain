# Captain 0.1.0-alpha.8

Captain's eighth public early-access release adds Captain Forge, a governed
runtime for readable native capabilities, and makes Codex subscription limits
truthful across every interactive surface. The durable state, supervised
restart, model-aware context, memory, and Telegram Rich Message contracts from
alpha.7 remain in place.

## Install

macOS, Linux, and VPS:

```bash
curl -fsSL https://github.com/Vivien83/captain/releases/download/v0.1.0-alpha.8/install.sh \
  | CAPTAIN_VERSION=v0.1.0-alpha.8 bash
```

Docker (`linux/amd64` and `linux/arm64`):

```bash
docker pull ghcr.io/vivien83/captain-agent-os:v0.1.0-alpha.8
```

The moving prerelease channel is
`ghcr.io/vivien83/captain-agent-os:alpha`. Pin the immutable version tag for a
reproducible install.

The release also contains an `x86_64-pc-windows-msvc` CLI zip. Every host
archive has a SHA-256 sidecar and platform manifest; `manifest.json` covers all
five targets.

## Captain Forge

- A reviewed `*.captain` TOML file placed in a global or project `.captain/`
  directory is compiled and hot-loaded as a typed `cap_*` tool without a
  daemon restart.
- CapSpecs compose Captain's existing tools into dependency-aware durable DAGs.
  Independent nodes may run concurrently; dependencies and mutations remain
  ordered.
- Every primitive step re-enters the central ToolRunner. The effective
  authority is the intersection of the caller, current policy, and the
  capability declaration, so a CapSpec cannot grant itself broader access.
- Revisions, exact-hash approvals, rejections, disablement, rollback, run
  deadlines, interruptions, and uncertain external mutations are persisted.
  Recovery after restart never blindly repeats an uncertain side effect.
- Authenticated API, Control, TUI, and Telegram surfaces let an operator review
  source and resolve exact pending decisions without asking the model to
  approve its own code.
- A process-level certification exercised 130 checks across 14 durable runs,
  including hot reload, project precedence, security denials, real tool
  execution, Control, TUI, Telegram, and an actual `SIGKILL` recovery. The
  public-safe evidence is in
  `docs/evidence/CAPSPEC1_REAL_CERTIFICATION_2026-07-18.md`.

## Subscription and internal quotas

- Captain's `max_llm_tokens_per_hour` is a durable rolling one-hour safety
  guard backed by SQLite. Restarting the daemon no longer clears the usage
  window, and a rejection is returned as structured HTTP `429` metadata rather
  than an opaque HTTP 500.
- For authenticated Codex subscriptions, Captain observes the official account
  usage response, dynamic `x-codex-*` headers, and `codex.rate_limits` stream
  events. It does not infer plan allowances from local token counters or a
  copied quota schedule.
- CLI status, TUI, the xterm Web terminal, Control, the retained desktop
  wrapper, `/api/status`, and `/api/budget` share the persisted provider
  snapshot. Missing data is `unavailable`; old data is `stale`; neither is
  called unlimited.
- Compact Chat status names the configured active model first. It draws gauges
  only for provider-wide windows and families matching that model; limits for
  another model remain visible as outside the active model without masquerading
  as its allowance. Status and Budget retain the complete provider report.
- When the provider reports exhaustion, Captain preserves the reported reset,
  avoids a futile retry, and never silently switches model or provider.

The immutable public source commit, 20-asset GitHub Release, and multi-platform
OCI index digest are filled from the live publication audit after the tag and

## Recovery boundary

The durability guarantee begins after the relevant tool or API reports success.
An external side effect still in flight at abrupt termination may require
inspection or an explicit idempotent retry. Backups remain necessary; no local
durability mechanism makes the host disk itself redundant.

## Known limitation

The explicit per-turn memory write opt-out remains incomplete in `alpha.8`.
Post-turn semantic graph, MemPalace, reflection, conversation-learning, and
workflow-learning writes are suppressed, but the core agent-loop finalizer
still stores its local episodic interaction fragment. This is separate from the
normal session transcript and mandatory audit that are intentionally retained.
Do not rely on the opt-out to prevent that local semantic fragment until a
later immutable release closes the core finalizer path.

## Alpha notice

This is a prerelease. Keep backups, review every installed CapSpec and granted
capability, and do not use it as the sole copy of critical data. macOS binaries
are ad-hoc signed but not Apple-notarized; the Windows CLI is not
Authenticode-signed. Verify the published SHA-256 sidecar and expect an explicit
operating-system approval on first launch.

Captain binds to loopback by default. For remote access, configure Captain
authentication and place it behind HTTPS; never expose an unauthenticated
daemon directly to the Internet.

The presentation site is deployed separately at `https://captainagent.fr/`.
Its source is intentionally not part of the public Git repository or release.
