# Captain 0.1.0-alpha.9

Captain's ninth public early-access release turns workflow learning into a
durable operator-controlled lifecycle and adds a native release monitor. The
Captain Forge, provider quota, crash recovery, memory, session, and Telegram
Rich Message contracts from alpha.8 remain in place.

## Install

macOS, Linux, and VPS:

```bash
curl -fsSL https://github.com/Vivien83/captain/releases/download/v0.1.0-alpha.9/install.sh \
  | CAPTAIN_VERSION=v0.1.0-alpha.9 bash
```

Docker (`linux/amd64` and `linux/arm64`):

```bash
docker pull ghcr.io/vivien83/captain-agent-os:v0.1.0-alpha.9
```

The moving prerelease channel is
`ghcr.io/vivien83/captain-agent-os:alpha`. Pin the immutable version tag for a
reproducible install.

The release also contains an `x86_64-pc-windows-msvc` CLI zip. Every host
archive has a SHA-256 sidecar and platform manifest; `manifest.json` covers all
five targets.

## Durable workflow learning

- Captain records real workflow episodes at the Kernel and ToolRunner boundary
  instead of counting sliding tool-name fragments.
- Evidence is grouped deterministically and classified as memory, refinement,
  Skill, readable native CapSpec, Automation, or no proposal. Generated drafts
  use Captain's configured active model and cannot request tools outside the
  observed canonical workflow.
- Drafts remain inert while Captain validates and tests them in isolated native
  registries. Installation, canary, activation, failure, and rollback are
  journaled and recoverable after an abrupt restart.
- Skills, CapSpecs, Automations, and refinements use one exact operator card and
  decision version across Telegram, TUI, Control Web, Desktop, and the
  authenticated API. Stale cards and fabricated revisions fail closed.
- The SkillSynthesizer v3.13 detector, proposer, policy, diff, writer, settings,
  and agent-facing decision path are retired. Historical rows are archived and
  legacy REST routes return an actionable `410 Gone` response.
- Real-domain tests cover sourced technology research, household document
  extraction, recurring VPS reporting, memory-only preferences, and recovery
  from a power loss after model generation without a second model call.

## Native release monitor

- Captain checks its compatible official release channel after startup and
  every 12 hours. Stable installations never opt into prereleases; a
  prerelease may advance through newer prereleases and then to stable.
- A self-updating host receives a proposal only when its exact platform archive
  and SHA-256 sidecar both exist.
- The explicitly authorized Telegram operator receives a Rich card with
  **Update**, **Defer 24 h**, and **Refuse this version**. The exact callback is
  resolved before session restoration or model dispatch.
- Candidate, decision, detached installer result, and Telegram delivery lease
  survive restart. Orphaned attempts recover after a bounded timeout, malformed
  results are quarantined, and dead delivery is reopened without duplicating a
  card already delivered.
- Docker and unsupported host modes remain operator-owned. Captain never gains
  the host Docker socket and keeps the manual procedure observable until a
  later check sees the new runtime.
- `captain status` and `GET /api/status` expose the release cadence, candidate,
  attempt, result, and notification state under `runtime_update`.

## Publication provenance

The immutable public source commit, annotated tag object, 20-asset GitHub
Release, multi-platform OCI index digest, anonymous download/pull evidence, and
zero-Actions proof are filled from the live publication audit after those
surfaces exist. They are never copied from an older release.

## Recovery boundary

The durability guarantee begins after the relevant tool or API reports success.
An external side effect still in flight at abrupt termination may require
inspection or an explicit idempotent retry. Backups remain necessary; no local
durability mechanism makes the host disk itself redundant.

## Known limitation

The explicit per-turn memory write opt-out remains incomplete in `alpha.9`.
Post-turn semantic graph, MemPalace, reflection, conversation-learning, and
workflow-learning writes are suppressed, but the core agent-loop finalizer
still stores its local episodic interaction fragment. This is separate from the
normal session transcript and mandatory audit that are intentionally retained.
Do not rely on the opt-out to prevent that local semantic fragment until a
later immutable release closes the core finalizer path.

## Alpha notice

This is a prerelease. Keep backups, review every installed CapSpec, Skill, and
Automation plus their granted capabilities, and do not use Captain as the sole
copy of critical data. macOS binaries are ad-hoc signed but not
Apple-notarized; the Windows CLI is not Authenticode-signed. Verify the
published SHA-256 sidecar and expect an explicit operating-system approval on
first launch.

Captain binds to loopback by default. For remote access, configure Captain
authentication and place it behind HTTPS; never expose an unauthenticated
daemon directly to the Internet.

The presentation site is deployed separately at `https://captainagent.fr/`.
Its source is intentionally not part of the public Git repository or release.
